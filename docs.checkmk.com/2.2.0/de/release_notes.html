Update auf Version 2.2.0 Update-Matrix für Version 2.2.0 Grundsätzliches zur Installation von Checkmk Linux überwachen Windows überwachen

1. Einleitung

Dieses Dokument listet die unterstützten Plattformen und Integrationen von Checkmk 2.2.0 auf. Sofern keine besonderen Gründe vorliegen (z. B. Sicherheit), werden diese Plattformen während der aktiven Pflege im Checkmk 2.2.0 Produkt-Lifecycle einschließlich aller Patch-Versionen unterstützt.

2. OS für den Server

Checkmk kann auf den folgenden Linux-Server-Betriebssystemdistributionen und -versionen installiert werden, die auf x86_64-Hardware laufen. Dazu geben wir eine Liste der unterstützen Versionen heraus. Einträge können folgende Zusätze enthalten:

  • Deprecated: So gekennzeichnete LTS/LTSS Versionen werden noch während des gesamten Lebenszyklus von Version 2.2.0 unterstützt und fallen danach aus dem Support heraus. Bei den Ubuntu STS Versionen endet die Bereitstellung von Checkmk 2.2.0 Paketen bereits kurz nach Einstellung der Unterstützung mit Updates durch Ubuntu.

  • Once released: Die Unterstützung für diese Versionen ist geplant. Die Bereitstellung von Checkmk 2.2.0 Paketen erfolgt zeitnah nach Erscheinen der neuen Distributionsversion.

Wir werden auch neue Versionen dieser Distributionen für die Unterstützung evaluieren. Diese entnehmen Sie bitte der Update-Matrix.

Wichtig: Ab Checkmk 2.3.0 werden STS-Versionen von Ubuntu generell nicht mehr unterstützt.

3. Browser

Die Checkmk-Benutzeroberfläche ist für die folgenden Desktop-Browser und Mindestversionen geprüft:

  • Google Chrome >= 100 unter Windows, Linux und macOS

  • Firefox >= 100 unter Windows, Linux und macOS

  • Microsoft Edge >= 100 unter Windows

  • Safari >= 15.6.1 unter macOS

Die mobile Checkmk-Benutzeroberfläche ist für die folgenden mobilen Browser geprüft:

  • Safari unter iOS (aktuelle und letzte Hauptversion des Betriebssystems)

  • Google Chrome unter Android (aktuelle und letzte Hauptversion des Betriebssystems)

4. OS für den Checkmk-Agenten

Der Checkmk-Agent kann auf den folgenden Betriebssystemen installiert und betrieben werden, wenn grundlegende Anforderungen erfüllt sind.

4.1. Linux/Unix

Alle Linux- und Unix-Betriebssysteme werden unterstützt, sofern sie mindestens die folgenden technischen Komponenten in der minimalen Version mitbringen:

  • Bash >= 4.2

  • ksh >= 93

  • Perl >= 5.14.0

Zusätzlich muss eine der folgenden Komponenten vorhanden sein:

  • systemd >= 219

  • xinetd >= alle Versionen

Bitte beachten Sie: Es kann sein, dass Plugins für ihre Funktion zusätzliche Komponenten benötigen. Python-Plugins erfordern die Version 2.7 oder > 3.5.

4.2. Windows

Alle folgenden 64-bit-Versionen der Windows Betriebssysteme werden unterstützt:

  • 2012

  • 2012 R2

  • 2016

  • 2019

  • 2022

Für ältere Systeme stellen wir einen Legacy Agenten bereit. Dieser hat ein geringeres Feature-Set und wird nicht offiziell unterstützt.

5. Grafana Plugin

Checkmk 2.2.0 unterstützt die folgenden Grafana Plugin Versionen:

  • >= 3.0.0

6. Checkmk Appliance

Checkmk 2.2.0 kann auf den folgenden Versionen der Checkmk Appliance ausgeführt werden:

  • >= 1.6.3

7. LDAP-Server

Checkmk 2.2.0 unterstützt die Synchronisierung mit LDAP der folgenden LDAP-Server:

  • Microsoft Active Directory

  • OpenLDAP

  • 389 Directory Server

8. SAML IdP

Die SAML-Integration unterstützt die folgenden Identity Provider (IdPs):

  • Microsoft Azure Active Directory

  • Google Cloud

  • Okta

Checkmk arbeitet möglicherweise auch mit anderen SAML-Integrationen zusammen, aber für diese kann keine Unterstützung gegeben werden.

9. ntopng

Die ntopng-Integration unterstützt die folgenden Versionen von ntopng Professional und Enterprise:

  • >= 5.6

10. Nagvis

Nagvis hängt von PHP ab und benötigt eine Mindestversion, um zu laufen. Stellen Sie sicher, dass Sie eine passende PHP-Version in Ihrer Installation haben, um Nagvis auszuführen:

  • >= 7.2

11. InfluxDB

Checkmk unterstützt den Export von Daten an die folgenden Versionen der InfluxDB API:

  • >= 2.0
